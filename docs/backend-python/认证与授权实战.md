# 认证与授权实战

Agent 的 API 接口暴露在公网上，没有认证层等于裸奔。JWT、OAuth 2.0、API Key 三种模式各有适用场景，这篇文章帮你理清选型思路并给出 FastAPI 的完整实现。

## 三种认证模式对比

在动手写代码前，先理解三种模式的核心差异：

| 模式 | 适用场景 | 安全性 | 实现复杂度 |
|------|---------|--------|-----------|
| JWT | 第一方客户端（自家 Web/App） | 中（依赖 Token 有效期） | 低 |
| OAuth 2.0 | 第三方接入 / 单点登录 | 高（支持 Scope 细粒度控制） | 高 |
| API Key | M2M（Machine-to-Machine）/ CLI 工具 | 中（依赖 Key 管理） | 低 |

### 选型决策树

```
你的 API 给谁用？
├── 自家前端 / 移动端 → JWT
├── 第三方开发者 / 合作伙伴
│   ├── 需要用户授权 → OAuth 2.0
│   └── 服务间调用 → API Key
└── 内部微服务 → API Key（配合 Service Mesh 或 mTLS）
```

Agent 场景中三种模式都会用到：前端对话 UI 用 JWT，外部 IDE 插件接入用 OAuth 2.0，内部 Agent 间通信用 API Key。

## JWT 实战

### Token 结构

JWT 由三部分组成：Header、Payload、Signature。Payload 中存放用户身份和元数据：

```python
from datetime import datetime, timedelta, timezone
from jose import jwt

SECRET_KEY = "your-secret-key-here"  # 来自环境变量
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30


def create_access_token(data: dict, expires_delta: timedelta | None = None):
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + (expires_delta or timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES))
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
```

### FastAPI 集成

```python
from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials

security = HTTPBearer()


async def get_current_user(credentials: HTTPAuthorizationCredentials = Depends(security)):
    token = credentials.credentials
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        user_id: str = payload.get("sub")
        if user_id is None:
            raise HTTPException(status_code=401, detail="Invalid token")
        return user_id
    except jwt.JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")


@app.post("/api/v1/chat")
async def chat(user_id: str = Depends(get_current_user)):
    # user_id 已通过 JWT 认证
    ...
```

### Token 刷新策略

Access Token 有效期短（15-30 分钟），Refresh Token 有效期长（7-30 天）。用 Refresh Token 换取新的 Access Token：

```python
REFRESH_TOKEN_EXPIRE_DAYS = 7


def create_refresh_token(data: dict):
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + timedelta(days=REFRESH_TOKEN_EXPIRE_DAYS)
    to_encode.update({"exp": expire, "type": "refresh"})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)


@app.post("/api/v1/auth/refresh")
async def refresh_token(refresh_token: str):
    try:
        payload = jwt.decode(refresh_token, SECRET_KEY, algorithms=[ALGORITHM])
        if payload.get("type") != "refresh":
            raise HTTPException(status_code=400, detail="Invalid token type")
        user_id = payload.get("sub")
        new_access_token = create_access_token({"sub": user_id})
        return {"access_token": new_access_token, "token_type": "bearer"}
    except jwt.JWTError:
        raise HTTPException(status_code=401, detail="Invalid refresh token")
```

## OAuth 2.0 实战

Agent 平台接入第三方工具（Google Drive、Slack、GitHub）时，OAuth 2.0 是标准方案。以 Authorization Code Flow 为例：

```python
from authlib.integrations.starlette_client import OAuth

oauth = OAuth()
oauth.register(
    name="github",
    client_id="GITHUB_CLIENT_ID",
    client_secret="GITHUB_CLIENT_SECRET",
    authorize_url="https://github.com/login/oauth/authorize",
    authorize_params=None,
    access_token_url="https://github.com/login/oauth/access_token",
    redirect_uri="https://your-agent.com/auth/callback",
    client_kwargs={"scope": "repo user"},
)


@app.get("/auth/login")
async def login(request: Request):
    redirect_uri = request.url_for("auth_callback")
    return await oauth.github.authorize_redirect(request, redirect_uri)


@app.get("/auth/callback")
async def auth_callback(request: Request):
    token = await oauth.github.authorize_access_token(request)
    # 存储 token，关联到当前用户
    user_id = token.get("userinfo", {}).get("sub", "unknown")
    await db.store_oauth_token(user_id=user_id, provider="github", token=token)
    return {"message": "授权成功"}
```

## API Key 实战

内部服务间通信用 API Key 更轻量：

```python
import secrets
from fastapi import Header, HTTPException


def generate_api_key():
    return f"ask_{secrets.token_urlsafe(32)}"


async def verify_api_key(x_api_key: str = Header(None)):
    if x_api_key is None:
        raise HTTPException(status_code=401, detail="Missing API Key")
    # 从数据库或缓存中验证
    key_info = await db.get_api_key(x_api_key)
    if key_info is None:
        raise HTTPException(status_code=401, detail="Invalid API Key")
    if key_info["expires_at"] and key_info["expires_at"] < datetime.now(timezone.utc):
        raise HTTPException(status_code=401, detail="API Key expired")
    return key_info


@app.post("/api/v1/internal/agent/task")
async def create_task(key_info: dict = Depends(verify_api_key)):
    return {"status": "ok"}
```

## 安全最佳实践

1. **密钥管理**：JWT Secret 和 API Key 用环境变量或 Secret Vault，不要硬编码
2. **速率限制**：认证接口比普通接口更严格的限流策略
3. **审计日志**：所有认证事件（登录、Token 刷新、权限变更）都记录结构化日志
4. **Token 吊销**：维护 Token 黑名单（Redis Set 结构），允许强制登出
5. **HTTPS Only**：生产环境强制 HTTPS，防止 Token 中间人截获
6. **最小 Scope**：OAuth 和 API Key 遵循最小权限原则，按需授权

## 总结

认证与授权是 Agent 后端安全的第一道防线。JWT 负责第一方用户认证，OAuth 2.0 处理第三方集成，API Key 服务内部通信。在实现时，重点关注密钥安全、Token 生命周期管理和审计日志，这三者是最容易出问题的环节。
